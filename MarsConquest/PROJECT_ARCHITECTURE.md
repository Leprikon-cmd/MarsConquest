# Mars LogBook - архитектура

Последнее обновление: 2026-07-14

Mars LogBook - iOS-компаньон для подсчёта очков, сохранения партий и просмотра статистики настольной игры «Покорение Марса». Приложение работает локально, без сервера.

## Слои приложения

1. **SwiftUI-интерфейс**: экраны новой партии, подсчёта, статистики и настроек.
2. **Локальная модель партии**: `LocalGameData` хранит ввод пользователя до нажатия «Сохранить результаты».
3. **Менеджеры**: `ScoreManager` считает очки, `GameSaver` переносит завершённую партию в Core Data, `CoreDataManager` открывает локальную базу.
4. **Core Data**: постоянное хранилище партий, игроков, очков, наград, достижений, профилей и справочников.

## Главный сценарий

```text
ContentView -> AddPlayersView -> AddPlayerScreen -> ScoreScreen
                                                    |
                                                    v
                                                GameSaver
                                                    |
                                                    v
                                                Core Data
                                                    |
                                                    v
                                  StatisticsScreen / GameDetailView
```

`ScoreScreen` работает только с `LocalGameData`. После успешного сохранения экран закрывается, а пользователь открывает уже постоянную сущность `Game`.

## Локальная модель партии

`LocalGameData` содержит дату, игровое поле, поколение, включённые дополнения, игроков, выбранные достижения и награды.

- `LocalPlayer`: имя, цвет, корпорация, два пролога и категории очков.
- `LocalAchievement`: название и UUID единственного игрока, первым заявившего достижение.
- `LocalAward`: название, массив первых мест и массив вторых мест для ничьих.

## Правила подсчёта

`ScoreManager` складывает базовые категории очков и бонусы:

- достижение: `+5` победителю;
- награда, первое место: `+5` каждому игроку при ничьей;
- второе место: `+2`, только при трёх и более игроках, единственном первом месте и без пересечения с ним;
- политика учитывается только при включённом дополнении «Кризис».

## Core Data

Главные сущности:

- `Game`: дата, поле, поколение, дополнения, игроки, награды и достижения;
- `Player`: участник конкретной партии;
- `Score`: категории очков игрока;
- `Achievement` и `Award`: присуждённые бонусы в конкретной партии;
- `SavedPlayer`: постоянный профиль игрока;
- `Corporation`, `Prologue`, `AchievementTemplate`, `AwardTemplate`: справочники выбора.

Модель Core Data имеет две версии. Текущая - `GameDataModel 2`: в ней количество поколений хранится только в `generation`. Предыдущая версия сохранена исключительно для автоматической миграции уже существующих баз.

## Справочники

`GameData.swift` хранит встроенный каталог корпораций, прологов, наград и достижений. При запуске `InitialDataLoader` синхронизирует его с Core Data: добавляет недостающие записи и убирает точные дубликаты.

Все экраны выбора читают справочники из Core Data. Поэтому у интерфейса один рабочий источник данных, а `GameData` нужен только для начального наполнения и обновления каталога.

## Технические соглашения

- Экран закрывается через современный SwiftUI `dismiss()`.
- Навигация строится на `NavigationStack` и `navigationDestination`.
- Повторное быстрое нажатие «Сохранить результаты» заблокировано состоянием `isSaving`.
- Открытые продуктовые и технические задачи ведутся в `TASKS.md`.
