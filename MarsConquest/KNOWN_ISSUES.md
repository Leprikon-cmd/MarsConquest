MarsConquest

------------------------------------------------
1. КРИТИЧЕСКИЕ ЛОГИЧЕСКИЕ ПРОБЛЕМЫ
------------------------------------------------

### 1.1 Награды и достижения не сохраняются

Экран ScoreScreen позволяет назначать:
- награды
- достижения

Но назначения хранятся только в:

assignedAchievements: [String: UUID]
assignedAwards: [String: UUID]

Эти данные НЕ передаются в GameSaver и НЕ сохраняются в CoreData.

Фактически пользователь может назначить бонусы, но они теряются при сохранении игры.

Затронутые файлы:

ScoreScreen.swift  
GameSaver.swift


------------------------------------------------

### 1.2 Награды и достижения представлены строками

В LocalGameData:

achievements: [String]  
awards: [String]

Но в проекте уже существуют модели:

LocalAchievement  
LocalAward  

И сущности CoreData:

Achievement  
Award  

Получается три разных слоя данных.

Это главный архитектурный разрыв проекта.


------------------------------------------------

### 1.3 Итоговые очки не учитывают награды и достижения

ScoreManager считает итог:

terraformingRating  
greenery  
cities  
victoryPoints  
resourcesOnCards  
conditionsOnCards  
politics

Но:

- награды
- достижения

в расчёт не входят.

В результате:

итоговый счёт на экране может отличаться от реального счёта партии.


------------------------------------------------

### 1.4 Модель наград и достижений не поддерживает правила игры

Текущая логика:

achievement → один победитель  
award → один победитель

Но правила игры:

Достижения:
- при игре на двоих возможна ничья
- оба получают 5 ПО

Награды:
- 1 место → 5 ПО
- 2 место → 2 ПО
- возможна ничья за первое место
- при ничьей второе место не даётся

Текущая модель данных не поддерживает это корректно.

2. НЕСООТВЕТСТВИЕ МОДЕЛЕЙ ДАННЫХ
------------------------------------------------

### 2.1 duration vs generation

В GameInfoView используется:

duration

Но отображается как:

"Поколение"

В CoreData присутствуют оба поля:

duration  
generation

Неясно какое поле должно использоваться.


------------------------------------------------

### 2.2 player.scoreValue не используется

В модели Player есть поле:

scoreValue

Но:

- оно нигде не записывается
- нигде не используется

Это либо устаревшее поле, либо недоделанная логика.


------------------------------------------------

### 2.3 порядок игроков не гарантирован

В GameDetailView:

game.players?.allObjects as? [Player]

CoreData не гарантирует порядок.

Игроки могут отображаться в случайном порядке.


------------------------------------------------
3. АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ
------------------------------------------------

### 3.1 ScoreManager смешивает UI и логику

ScoreManager содержит:

- UI (SwiftUI View)
- расчёт очков
- генерацию номера игры

Это разные уровни архитектуры.


------------------------------------------------

### 3.2 Дублирование справочников

Справочные данные находятся одновременно:

GameData.swift  
CoreData templates  
UI

Например:

цвета игроков  
карты Марса  
награды  
достижения


------------------------------------------------

### 3.3 InitialDataLoader не защищён от повторного запуска

generateInitialGameData()

не проверяет существование записей.

Повторный вызов создаст дубликаты.


------------------------------------------------
4. ПРОБЛЕМЫ UI / UX
------------------------------------------------

### 4.1 ScoreTableView позволяет вводить отрицательные значения

Нет валидации числовых полей.


------------------------------------------------

### 4.2 клавиатура numberPad не имеет кнопки "Готово"

На iPhone пользователь не может закрыть клавиатуру.


------------------------------------------------

### 4.3 список игроков отображает только цвет

В таблице очков игроки обозначены только кружками.

При 4–5 игроках это может быть неочевидно.


------------------------------------------------

### 4.4 форма AddPlayerScreen допускает имя из пробелов

name.isEmpty не проверяет строки вида:

"     "


------------------------------------------------
5. ПРОБЛЕМЫ КОДА
------------------------------------------------

Используются одновременно:

NavigationView  
NavigationStack

### 5.2 устаревший presentationMode

В новых версиях SwiftUI рекомендуется использовать:

dismiss()

### 5.4 магические строки

"Эллада"

лучше заменить на enum.


------------------------------------------------
6. ПРОБЛЕМЫ СТАТИСТИКИ
------------------------------------------------

StatisticsScreen фактически не показывает статистику.

Есть только:

- количество игр
- список игр

Но база уже позволяет считать:

- победы игроков
- статистику корпораций
- статистику прологов
- средний счёт
