# MarsConquest — контекст проекта

## Назначение
MarsConquest — приложение для:
- подсчёта победных очков в настольной игре Terraforming Mars;
- сохранения сыгранных партий;
- накопления статистики игроков;
- возможного будущего перехода к серверной общей статистике.

Главная идея проекта:
1. быстро и удобно заносить результаты партии;
2. сохранять полную историю игр;
3. видеть статистику по игрокам, корпорациям, прологам и партиям.

## Технологии
- SwiftUI
- CoreData
- iOS

## Текущее состояние
Уже реализовано:
- структура проекта по папкам;
- CoreData-модель;
- сохранение игр;
- сохранение игроков;
- сохранение очков;
- базовые экраны;

## Структура проекта

### Data
Слой данных и подготовки локальных данных.

- `GameData` — рабочие игровые данные / модели / доступ к данным.
- `GameDataModel` — модель данных проекта.
- `InitialDataLoader` — начальная загрузка справочников и шаблонов.
- `LocalGameData` — локальные справочные данные игры.

### Managers
Логика и сервисные менеджеры.

- `CoreDataManager` — настройка и работа с CoreData.
- `GameSaver` — сохранение игры и связанных сущностей.
- `ScoreManager` — расчёт и агрегация очков.

### Screens
Крупные экраны приложения.

- `AddPlayerScreen` — экран добавления игроков.
- `ScoreScreen` — экран ввода/редактирования очков.
- `StatisticsScreen` — экран статистики.

### Views
Составные представления и вспомогательные экраны.

- `AchievementsListView` — выбор достижений.
- `AddPlayersView` — UI для добавления игроков.
- `AwardsListView` — выбор наград.
- `GameDetailView` — экран деталей сохранённой игры.
- `GameInfoView` — экран параметров игры / создания новой игры.
- `ScoreTableView` — таблица ввода очков игроков.

### Корневые и служебные файлы

- `Assets` — ресурсы приложения.
- `ColorButton` — кнопка выбора цвета.
- `ContentView` — корневой контейнер интерфейса.
- `MarsConquestApp` — точка входа приложения.
- `ScoreTextField` — кастомное поле ввода чисел/очков.

## Смысловая модель приложения

Основные сущности:
- Игра (`Game`)
- Игрок (`Player`)
- Очки (`Score`)
- Награда (`Award`)
- Достижение (`Achievement`)
- Шаблон награды (`AwardTemplate`)
- Шаблон достижения (`AchievementTemplate`)
- Корпорация (`Corporation`)
- Пролог (`Prologue`)

## Основной пользовательский сценарий
1. Создать новую игру.
2. Добавить игроков.
3. Указать параметры партии.
4. Ввести очки игроков.
5. Выбрать награды и достижения.
6. Сохранить игру.
7. Позже открыть детали игры и статистику.

## Логические замечания
- Награды и достижения хранятся отдельно от основных очков.
- Нужна явная и прозрачная логика начисления очков за награды и достижения.
- Нужен понятный UI для выбора победителей/мест.
- Корпорации и прологи сейчас частично заданы строками у игрока, а частично отдельными сущностями статистики.

1. Не переписывать проект целиком без необходимости.
