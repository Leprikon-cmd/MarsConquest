# Mars LogBook - инвентаризация текстов интерфейса

Последнее обновление: 2026-07-16

Этот документ готовит локализацию интерфейса. Он не меняет игровые названия,
которые пока сохраняются как текст и будут переноситься на стабильные ключи отдельным этапом.

## Правила тона

- Интерфейс должен быть понятен без знания русских шуток и терминов.
- Атмосфера экспедиции уместна в названиях экранов и главном действии, но не в ошибках,
  настройках и обычных формах.
- Короткие подписи предпочтительнее сокращений: `Бонусные очки` понятнее, чем `Доп. очки`.
- `CEO` уместен на экране добавления игрока: он задаёт роль главы корпорации.
  На плашке игрока в составе команды это повторение не нужно: там остаётся только имя.
- Перевод должен передавать контекст и тон, а не копировать русские слова буквально.
- Названия игры `Mars LogBook`, игровых полей, корпораций, прологов, колоний, наград и достижений
  не переводятся этим этапом.

## Тексты, которые можно оставить без стилистической правки

- `Корпорация`, `Прологи`, `Колонии`, `Награды`, `Достижения`.
- `Дата`, `Поле`, `Поколение`, `Рейтинг`, `Игра`, `Итог`, `Всего`.
- Сообщения валидации: они уже прямые и понятные; при переносе в каталог строк
  нужно только заменить вставки значений на форматируемые параметры.

## Что меняем первым после утверждения словаря

1. Создаём `Localizable.xcstrings` с русским и английским языками.
2. Переносим общие кнопки, навигацию, формы, диалоги и сообщения об ошибках.
3. Проверяем английский интерфейс на маленьком iPhone.
4. Только затем отдельно вводим стабильные ключи игрового справочника и переводим игровые названия.

## Зафиксированные решения для следующего изменения интерфейса

- Отдельный заголовок `Доп. очки: награды и достижения` убран: выбранные награды и достижения
  продолжают основную таблицу очков. Круг в ячейке означает доступное назначение, число показывает
  уже начисляемые очки.
- На плашке игрока в команде показывать только имя, без префикса `CEO`.
- На экране выбора места высадки добавить короткую подсказку о свайпе. Она должна появляться
  ненавязчиво и не мешать пользователю после первого знакомства с жестом.
